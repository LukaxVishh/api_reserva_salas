SALA LIVRE API - RESUMO DO PROJETO

1. REGRAS E EXIGÊNCIAS DO TRABALHO

O projeto deve desenvolver uma API REST completa aplicando:
- comunicação via HTTP
- manipulação de JSON
- separação de responsabilidades
- regras de negócio reais

Organização do trabalho:
- trabalho em equipe
- tema livre definido pelo grupo
- entrega do código + defesa presencial

Requisitos obrigatórios da aplicação:
- uso de Controllers
- separação em camadas
  - Controllers
  - Models
  - Repository
- CRUD mínimo para pelo menos uma entidade principal
  - GET (listar todos)
  - GET por ID
  - POST (criar)
  - PUT (atualizar)
  - DELETE (remover)

Uso de Java/JSON:
- uso correto de classes e objetos
- encapsulamento
- construtores
- collections (List, Set ou Map)
- tratamento de exceções
- pelo menos uma exceção personalizada
- uso de biblioteca JSON (Jackson)
- serialização (objeto para JSON)
- desserialização (JSON para objeto)

Regras de negócio:
- a API precisa ter processamento real
- não será aceito CRUD vazio sem lógica
- exemplos esperados:
  - cálculo
  - validação
  - classificação/status

Persistência de dados:
- uso obrigatório de banco de dados com JDBC
- implementar ao menos 1 repositório com repository pattern
- separar acesso a dados da lógica de negócio
- fluxo esperado do projeto:
  Controller -> Service -> Repository

2. TEMA ESCOLHIDO PELO GRUPO

Tema: Sistema de reserva de salas

Motivo da escolha:
- é um tema menos genérico
- é simples de implementar
- permite regras de negócio reais
- atende bem ao requisito de duas entidades relacionadas

3. ENTIDADES DO PROJETO

Entidade 1: Sala
Atributos planejados:
- id
- nome
- bloco
- capacidade
- temProjetor
- ativa

Entidade 2: Reserva
Atributos planejados:
- id
- nomeResponsavel
- descricao
- data
- horaInicio
- horaFim
- status
- salaId

Relacionamento:
- uma Sala pode ter várias Reservas
- uma Reserva pertence a uma Sala

4. REGRAS DE NEGÓCIO PLANEJADAS

- não permitir reserva em horário já ocupado
- validar que horaInicio seja menor que horaFim
- limitar duração máxima da reserva
- permitir reserva apenas em salas ativas
- validar campos obrigatórios
- classificar status da reserva

Status planejados para reserva:
- AGENDADA
- EM_ANDAMENTO
- FINALIZADA
